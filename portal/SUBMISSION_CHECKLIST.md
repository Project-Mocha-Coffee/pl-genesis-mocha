# PL_Genesis Submission Checklist – Project Mocha

## Required fields

- **Project name**
  - Project Mocha – Cross‑Chain Coffee Farm Vaults

- **One‑sentence description**
  - Real‑world coffee farms tokenized as ERC‑4626 vaults across Scroll, Base, and Starknet, with a consumer‑grade investor portal.

- **Hackathon tracks**
  - Primary: Crypto, Infrastructure & Digital Rights
  - Optional / extension: AI & Robotics, Agent Only, 8004, Starknet, Filecoin, Hypercerts (if integrated).

- **GitHub links**
  - Frontend portal: `https://github.com/Project-Mocha-Coffee/portal-main`
  - Smart contracts: `https://github.com/Project-Mocha-Coffee/smart-contracts`
    - Branches: `erc4626-scroll-base`, `erc4626-starknet`, `erc4626`

- **Demo link**
  - URL of deployed portal (e.g. Vercel) or a short Loom/YouTube demo.
  - Show: network selector, farm list, MBT balance, example investment flow, Starknet ICO flow.

- **Video (2–5 min) – outline**
  1. Problem + who it’s for (investors + farmers).
  2. High‑level architecture (multi‑chain vaults + portal).
  3. Live demo of the portal.
  4. Quick look at key contracts / deployment docs.
  5. Why it fits the chosen track(s) and what’s next.

- **Documentation**
  - Link `PL_GENESIS_SUMMARY.md` in the repo root or `docs/`.
  - Add a short `Track Fit` section for each track you submit to (copy from `TRACK_BLURBS.md`).

## Final review before submitting

- [ ] Repos are public.
- [ ] Branch names mentioned in the submission match GitHub.
- [ ] Environment variables for the demo (especially `NEXT_PUBLIC_STARKNET_ICO_ADDRESS`) are set.
- [ ] Video link works and is set to public/unlisted.
- [ ] Summary text fits character limits (keep a short and a long version handy).
